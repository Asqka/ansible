Любой playbook начинается с определения задачи (name), где просто описывается, что делает данный сценарий. Далее указывается hosts — это список серверов или групп серверов из inventory-файла, на которых будет выполняться автоматизация. После этого часто используется параметр become: yes, который означает выполнение задач с правами администратора (root), что необходимо для установки пакетов, настройки сервисов и изменения системных файлов.
Основная часть playbook — это tasks. Именно здесь описываются все действия, которые Ansible должен выполнить. Каждая задача начинается с - name, где даётся понятное описание действия. Затем используется модуль, который определяет тип операции. Например, модуль apt используется для установки пакетов, user — для создания пользователей, copy — для копирования файлов, lineinfile — для изменения строк в файлах, а serviceили command — для управления сервисами.
Чаще всего playbook работает по принципу последовательного выполнения: задачи выполняются сверху вниз, и каждая следующая зависит от предыдущей. Например, сначала устанавливается программа, затем она запускается, а потом настраивается.
Также важно понимать, что Ansible работает через SSH и не требует установки агентов на сервере. Он просто подключается к целевой машине, выполняет задачи и возвращает результат выполнения, который отображается в терминале в виде ok, changed или failed.
Таким образом, стандартный playbook всегда состоит из: описания (name), выбора серверов (hosts), повышения прав (become), списка задач (tasks) и модулей, которые выполняют конкретные действия.
